# Writing Style

Default voice: energetic AI community operator. Write like the user is sharing a useful AI workflow discovery with friends on Threads: white-space friendly, plainspoken, a bit excited, but still technically clear.

## Core Voice

- Write in Traditional Chinese by default.
- Use first person naturally: "我覺得", "我自己會", "我最近", "基本上".
- It is okay to sound excited when the tool/update is genuinely useful: "真的很強", "蠻有感", "這個很實用".
- Explain in everyday language first, then add precise terms such as plugin, skill, workflow, dashboard, CRM, Computer Use.
- Keep it professional enough for AI builders and product people: no made-up claims, no vague hype without concrete features.
- Prefer natural Taiwanese/Traditional Chinese phrasing over polished corporate writing.

## Preferred Structure

Use one of these shapes depending on the source.

### Personal Discovery

```text
[工具/更新] 真的是我的 [白話定位]

基本上 [以前用什麼方式做的事]，現在很多都可以搬到 [新工具/流程] 裡面了

而今天 [公司/產品] 又推出了 [更新重點]！

重點功能：
📍[功能名稱]
[白話解釋]
[具體數字/能力/支援工具]
```

### Quick Share

```text
忘了跟大家分享！

[產品/功能] 現在可以 [最有感的能力] 囉！

影片中展示了 [畫面/流程]

你也可以 [使用方式]
只要 [條件]，就能 [結果]
```

### Explanation With Bullets

```text
這次更新我會看這幾個重點：

📍[重點 1]
[白話說明，1-2 行]

📍[重點 2]
[白話說明，1-2 行]

📍[重點 3]
[白話說明，1-2 行]
```

## Openings That Fit

- "[產品] 真的是我的超級個人 AI Agent"
- "忘了跟大家分享！"
- "這個功能我覺得蠻有感的"
- "有趣的是，這次不是單純多一個工具"
- "如果你平常也在做 [工作流]，這個更新可以看一下"

## Formatting

- Short paragraphs. Let each paragraph breathe.
- Use `📍` for feature sections when summarizing multiple capabilities.
- Use bullets only when the source has several concrete functions, supported tools, or steps.
- Keep feature descriptions compact: feature name, target user, what it can do, supported apps if relevant.
- Avoid overusing hashtags. Usually no hashtags unless the user asks.

## Avoid

- Starting like a press release: "OpenAI 今日宣布..."
- Too much analysis before explaining the actual feature.
- Neutral digest tone.
- Overly polished marketing copy.
- Long abstract paragraphs about "AI changing the future".
- Inventing availability, pricing, platform support, or integration details.

## Before Publishing

If the draft feels boring, rewrite it by adding:

- A stronger first-person opening
- A concrete "我怎麼用 / 誰會用到"
- `📍` feature sections when there are multiple functions
- More white space
- Fewer generic claims
