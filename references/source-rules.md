# Source Rules

- Prefer primary sources: official websites, official blog posts, docs, papers, release notes, GitHub repos, model cards, or direct company/person announcements.
- If an X post or pasted post includes an external URL, open and review that URL before drafting.
- If the external URL is not official, look for the official/original source when feasible and distinguish it from secondary coverage.
- Summarize official source findings before writing the Threads draft: what launched/changed, who it is for, what it can do, supported tools/platforms, limitations, pricing, and availability when stated.
- If an X post links to an article or paper, inspect the linked source before summarizing the claim.
- If the source is a video, identify what is visible, what is spoken, and what is inferred. Do not treat inference as confirmed fact.
- Preserve original X URLs, article URLs, paper URLs, and downloaded media paths in the output notes.
- If source access fails, state what was available and draft from the available material only.
- Avoid reposting private, paywalled, or unclear-origin media as if it were cleared for reuse.
- For fast-moving AI news, verify current status before saying "latest", "just launched", "today", or similar time-sensitive wording.

## Minimum Source Note

For each draft, keep enough traceability for the user to check before posting:

```text
來源：
- X 原文：
- 外部連結：
- 影片/圖片：
- 未確認但可能需要補查：
```
